## Cumulative Tech Debt (acceptable for v2.1)

All items below are MED/LOW findings from per-phase REVIEWs that were deferred rather than fixed. None are BLOCKER/HIGH; none affect the v2.0.0 release tag.

### MEDIUM (carry-forward to v2.1+)

- **05-MED-01 (HERMES-05 brief vs MCP-bundle semantic divergence for `chatlytics_actions`)** -- Implementation follows MCP bundle (GET catalog) and exposes the brief's "generic dispatcher" intent as `chatlytics_dispatch`. Documented in 05-CONTEXT/PLAN/SUMMARY; HERMES-06 README clarifies both tools. No code action.
- **06-MED-01 (smoke does not exercise live Hermes plugin loader)** -- `scripts/smoke.sh` verifies entry-point discovery + import + pytest, but does NOT spin up the Hermes gateway and observe `register(ctx)` being invoked. Full gateway smoke needs a Chatlytics gateway endpoint -- operator-lock-blocked. Recommend a v2.1 "HERMES-07 / live-loader integration smoke" phase using a respx-mocked Chatlytics backend.
- **02-MED-02 (conftest fixture seeds platform_registry without teardown)** -- Session-scoped autouse fixture has no cleanup. Harmless for `pytest tests/` but unusual. Worth auditing if v2.1 tool-registration tests exercise the registry from a second angle.
- **04-MED-01 (`_keep_typing` async-cm shape divergence from base coroutine)** -- The override is `@contextlib.asynccontextmanager` instead of the base plain-coroutine shape. Intentional per phase brief; all callers are in-plugin. Worth documenting more loudly in v2.1 or proposing an upstream PR to make the base contract accept either shape.

### LOW (open, deferred)

- **02-LOW-02 (`send_typing` log flood at `logger.warning` on transport errors)** -- Original `send_typing` call path still uses `logger.warning`. HERMES-04 `_keep_typing` heartbeat correctly uses `logger.debug`; the original outbound `send_typing` may flood logs when the gateway is unhealthy. v2.1 should consolidate.
- **02-LOW-01 (`send()` silently drops metadata keys named like reserved fields)** -- T-HERMES-02-03 mitigation correct; cosmetic warning log would aid diagnostics.
- **02-LOW-03 (`get_chat_info` returns `{}` on error, indistinguishable from valid empty response)** -- Forced by upstream `BasePlatformAdapter` contract. Document in v2.1 when `is_user_authorized()` integration lands.
- **03-LOW-01 (`webhook_path` not validated at `__init__`)** -- Missing-leading-slash misconfig raises inside `connect()` and leaves adapter half-connected. Hardening pass.
- **04-LOW-02 (`filename` injected for URL-path documents -- gateway honor unconfirmed)** -- Needs Chatlytics gateway-team confirmation, not adapter-side fix.
- **04-LOW-03 (`_keep_typing` initial fire is sequential, delays wrapped body up to 30s on degraded gateway)** -- Convert to fire-and-forget if tool-handler latency profile shows the ceiling hits in practice.
- **05-LOW-01 (`_make_tool_handler` swallows `Exception` from `ctx.get_platform` silently)** -- Add DEBUG log to aid diagnosis of custom `PluginContext` failures.
- **05-LOW-02 (media-tool schemas accept any non-empty string for `chatId`, no JID format check)** -- Lenient by design (matches MCP bundle baseline). v2.1 can decide whether to mirror MCP bundle's `looksLikeJid` regex.
- **05-LOW-03 (`chatlytics_login` returns `success: True` even when `webhook_registered=False`)** -- MCP bundle treats it as a fail; current behavior is "API call succeeded". v2.1 UX decision.
- **06-LOW-01 (smoke re-installs hermes-agent on every run; ~45-60s)** -- Acceptable cost; pre-built docker image would help but adds cache-invalidation risk.
- **06-LOW-02 (`_keep_typing` initial-fire failure logged at DEBUG, invisible without verbose logging)** -- Bump FIRST-fire failure to WARNING while keeping subsequent heartbeats at DEBUG.

## Risks for v2.1+

- **Live-loader integration smoke (06-MED-01)** -- The strongest test gap. v2.0 smoke proves entry-point discoverability and import-time correctness but does not prove `register(ctx)` works against a real `PluginContext`. First production deployment will reveal any latent integration issue. Recommend a v2.1 phase that wires `hermes.gateway.bootstrap.load_plugins()` against a respx-mocked Chatlytics backend and asserts all 21 tools land on the in-memory registry.
- **`_keep_typing` async-cm shape (04-MED-01)** -- Diverges from upstream base coroutine contract. All callers today are in-plugin so the divergence is inert, but if upstream Hermes ever invokes `_keep_typing` on a chatlytics adapter via the base path, it will silently no-op. Either upstream PR (relax base contract) or rename our override to `keep_typing_block()` and ship a thin base-coroutine compat wrapper.
- **Blocking file I/O in `_resolve_media_url`** -- HERMES-06 fix wraps `open()+read()` in `asyncio.to_thread` (commit `5e00da9`), closing 04-MED-02 and 05-MED-02. Worth a concurrency regression test in v2.1 once tool handlers see bursty multi-MB uploads.
- **`hermes-agent==0.14.0` not on PyPI** -- Install vector still requires `pip install hermes-agent @ git+...@v2026.5.16`. Documented in README. When upstream publishes 0.14 to PyPI, smoke and install commands can simplify.
- **PyPI publish deferred** -- Manifest is ready (`python -m build && twine upload dist/*` is a 1-command op). Operator decision lock.
