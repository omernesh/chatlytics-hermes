## 3. Entrypoint scheme

Current `pyproject.toml:28`:
```toml
[project.entry-points."hermes.adapters"]
chatlytics = "chatlytics_adapter:ChatlyticsAdapter"
```

Upstream group name: **`hermes_agent.plugins`** (`.hermes-ref/hermes_cli/plugins.py:76` ŌåÆ `ENTRY_POINTS_GROUP = "hermes_agent.plugins"`). Documented at `.hermes-ref/website/docs/user-guide/features/plugins.md:101` ("`hermes_agent.plugins` entry_points").

**The current entry-point is wrong** ŌĆö Hermes will not discover the chatlytics adapter at runtime. However, this is **architecturally moot**: Hermes's plugin system loads tools/hooks/commands via a `register(ctx)` function on the entrypoint module (see `.hermes-ref/hermes_cli/plugins.py:847`). Platform adapters are NOT loaded via entry points in Hermes ŌĆö they are hardcoded in `gateway/run.py:_create_adapter()` (per phase 169 PLAN.md step 3b, 16 patch sites).

**Recommendation:** Delete the `[project.entry-points]` block from `integrations/hermes/pyproject.toml` entirely. It documents a non-existent integration point and confuses future readers. Adapter integration is via vendored copy at `gateway/platforms/chatlytics.py` + 16-site Hermes monorepo patch (phase 169 territory).
